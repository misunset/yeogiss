# 여기쓰 — 쓰레기통 찾기 랜딩페이지

서울시 쓰레기통 찾기 앱 **여기쓰**의 다운로드 유도 랜딩페이지입니다.

## 폴더 구성

- `index.html` — 배포용 **단일 파일**. 폰트·아이콘·디자인 시스템·로고까지 전부 인라인되어 오프라인/GitHub Pages에서 바로 열립니다. (직접 편집하지 말 것 — 아래 소스를 고치고 다시 번들하세요.)
- `여기쓰 랜딩페이지.dc.html` — **편집용 원본 소스**. 실제로 수정하는 파일입니다.
- `support.js` — 원본을 브라우저에서 직접 열 때 필요한 런타임.
- `_ds/` — 쓰레기통 찾기 디자인 시스템(색·타입·컴포넌트 토큰). 소스가 참조합니다.
- `uploads/yeogiss-logo.png` — 브랜드 로고.
- `figma/` — **Figma 추출 사양 문서**(색상 토큰·컴포넌트 베리언트). 디자인을 수정할 때 참고하는 기준 문서가 여기 모입니다. 아래 *Figma 사양 문서* 참고.

## 수정하는 법

1. `여기쓰 랜딩페이지.dc.html` 을 편집합니다.
2. 브라우저에서 그 파일을 직접 열면 결과가 바로 보입니다(`support.js`·`_ds`·`uploads` 가 같은 폴더에 있어야 함).
3. 배포용 `index.html` 은 소스를 수정한 뒤 다시 번들해 갱신하세요.

## GitHub Pages 배포

이 폴더를 저장소의 `yeogiss/` 경로에 올리면 `사용자명.github.io/yeogiss/` 에서 `index.html` 이 바로 서비스됩니다.

## Figma 사양 문서 (`figma/`)

Figma 파일(`wnFuCttvzSg2bLpQ5ESfez`)에서 추출한 기준 문서입니다. 디자인 수정 시 이 문서의 값을 따릅니다.

- `figma/여기쓰_color_tokens.md` — 색상 토큰 (브랜드·중립·시스템 컴·그라디언트).
- `figma/여기쓰_components_variants.md` — 컴포넌트별 Variant 목록 (Button, tab_container, BinPin, Toast 등).

**새 버전을 올릴 때:** 같은 파일명으로 `figma/` 안에 덮어쓰면 됩니다. 파일명이 다르면 이전 버전과 섞이지 않도록 기존 파일을 지우거나 이름을 맞춰주세요. 색상 토큰이 바뀌면 그 값이 소스(`여기쓰 랜딩페이지.dc.html`)의 `.page` 변수·`_ds` 토큰과 일치하는지 확인해 반영합니다.
